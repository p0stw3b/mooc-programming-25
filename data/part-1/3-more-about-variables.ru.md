Переменные нужны в программировании по разным причинам. В переменной можно хранить любую информацию, которая понадобится позже во время выполнения программы.

В Python переменные создаются так:

`variable_name = ...`

Здесь `...` обозначает значение, которое будет храниться в переменной.

Например, когда вы использовали `input`, чтобы прочитать строку от пользователя, вы сохраняли эту строку в переменную, а затем использовали переменную позже:

```python
name = input("What is your name? ")
print("Hi, " + name)
```

<sample-output>

What is your name? **Ghosty**
Hi, Ghosty

</sample-output>

Значение, которое хранится в переменной, можно определить и через другие переменные:

```python
given_name = "Paul"
family_name = "Python"

name = given_name + " " + family_name

print(name)
```

<sample-output>

Paul Python

</sample-output>

Здесь значения в трёх переменных не получены из ввода пользователя. Они будут одинаковыми при каждом запуске программы. Это называют _жёстким кодированием_ (hard-coding) данных в программе.

## Изменение значения переменной {#changing-the-value-of-a-variable}

Как и следует из самого названия, значение, которое хранится в переменной, может меняться. В предыдущем разделе мы видели, что новое значение заменяет старое.

Во время выполнения следующей программы переменная `word` будет иметь три разных значения:

```python
word = input("Please type in a word: ")
print(word)

word = input("And another word: ")
print(word)

word = "third"
print(word)
```

<sample-output>

Please type in a word: **first**
first
And another word: **second**
second
third

</sample-output>

Значение в переменной меняется каждый раз, когда переменной присваивается новое значение.

Новое значение переменной можно вывести из её старого значения. В следующем примере `word` сначала получает значение из ввода пользователя, а затем получает новое значение — старое значение с тремя восклицательными знаками в конце:

```python
word = input("Please type in a word: ")
print(word)

word = word + "!!!"
print(word)
```

<sample-output>

Please type in a word: **test**
test
test!!!

</sample-output>

<text-box variant="hint" name="Как выбрать хорошее имя переменной">

* Обычно полезно называть переменные по их назначению. Например, если в переменной хранится слово, имя `word` — лучше, чем, скажем, `a`.

* В Python нет жёсткого ограничения на длину имени переменной, но есть другие ограничения. Имя должно начинаться с буквы и может содержать только буквы, цифры и символ подчёркивания &#95;.

* Строчные и заглавные буквы — разные символы. Переменные `name`, `Name` и `NAME` — это три разные переменные. У этого правила есть исключения, но пока мы их игнорируем.

* В Python распространена практика использовать в именах переменных только строчные буквы. Если имя состоит из нескольких слов, между словами ставят подчёркивание. У этого правила тоже есть исключения, но пока мы их игнорируем.

</text-box>

## Целые числа (integers) {#integers}

До этого момента мы хранили в переменных только строки, но позже нам будут нужны и другие типы данных. Сначала посмотрим на целые числа (integers). Целые числа не имеют дробной части, например `-15`, `0` и `1`.

Следующая программа создаёт переменную `age`, которая хранит целое число:

```python
age = 24
print(age)
```

Программа выводит:

<sample-output>

24

</sample-output>

Обратите внимание: здесь нет кавычек. Если бы мы добавили кавычки вокруг числа, переменная перестала бы быть целым числом и стала бы строкой. В строке тоже могут быть цифры, но обрабатываться она будет по-другому.

Почему вообще важно, какой тип у переменной, если следующая программа всё равно выводит одно и то же два раза?

```python
number1 = 100
number2 = "100"

print(number1)
print(number2)
```

<sample-output>

100
100

</sample-output>

Типы важны, потому что разные операции по-разному работают с разными типами переменных. Посмотрим на пример:

```python
number1 = 100
number2 = "100"

print(number1 + number1)
print(number2 + number2)
```

Вывод будет таким:

<sample-output>

200
100100

</sample-output>

Для целых чисел оператор `+` означает сложение, а для строк — конкатенацию, то есть «склеивание».

Не все операторы доступны для всех типов. Числа можно делить оператором `/`, но попытка разделить строку на число приведёт к ошибке:

```python
number = "100"
print(number / 2)
```

<sample-output>
TypeError: unsupported operand type(s) for /: 'str' and 'int'
</sample-output>

## Объединение значений при печати {#combining-values-when-printing}

Точно так же следующая программа не будет работать, потому что `"The result is "` и `result` имеют разные типы:

```python
result = 10 * 25
# следующая строка приводит к ошибке
print("The result is " + result)
```

Программа ничего не выводит, а вместо этого выбрасывает ошибку:

<sample-output>

TypeError: unsupported operand type(s) for +: 'str' and 'int'

</sample-output>

Python сообщает нам, что «просто так» объединить значения разных типов нельзя. Здесь `"The result is "` — строка, а значение в `result` — целое число.

Если мы всё-таки хотим напечатать строку и целое число в одной команде, число можно привести к строке функцией `str`, а затем объединить две строки обычным образом. Например, так:

```python
result = 10 * 25
print("The result is " + str(result))
```

<sample-output>

The result is 250

</sample-output>

У `print` есть встроенные возможности, которые позволяют выводить значения разных типов. Самый простой способ — поставить запятую между значениями: все значения будут напечатаны независимо от их типа:

```python
result = 10 * 25
print("The result is", result)
```

<sample-output>

The result is 250

</sample-output>

Обратите внимание: между значениями, разделёнными запятой, автоматически вставляется пробел.

## Печать с помощью f-строк {#printing-with-f-strings}

Что делать, если нам нужно больше контроля над тем, что и как печатать? _F-строки_ (f-strings) — это удобный способ форматировать вывод в Python. Синтаксис поначалу может выглядеть немного странно, но на практике f-строки часто оказываются самым простым решением.

С f-строками предыдущий пример выглядел бы так:

```python
result = 10 * 25
print(f"The result is {result}")
```

Разберёмся. В самом начале строки стоит буква `f`. Она говорит Python, что перед нами f-строка. Внутри строки, в фигурных скобках, находится имя переменной `result`. Её значение подставляется внутрь печатаемой строки. Вывод будет таким же, как и в предыдущих примерах:

<sample-output>

The result is 250

</sample-output>

Одна f-строка может содержать несколько переменных. Например, этот код:

```python
name = "Mark"
age = 37
city = "Palo Alto"
print(f"Hi {name}, you are {age} years old. You live in {city}.")
```

выводит:

<sample-output>

Hi Mark, you are 37 years old. You live in Palo Alto.

</sample-output>

Такой вывод трудно получить точно таким же, используя запятые в `print`. Например, программа

```python
name = "Mark"
age = 37
city = "Palo Alto"
print("Hi", name, ", you are", age, "years old. You live in", city, ".")
```

печатает:

<sample-output>

Hi Mark , you are 37 years old. You live in Palo Alto .

</sample-output>

Обратите внимание на автоматически добавленные пробелы между частями, разделёнными запятыми. Технически можно запретить `print` добавлять лишние пробелы, но это не стоит усилий, поскольку вместо этого можно использовать f-строки.

В своей простоте «запятая» в `print` иногда бывает полезна, но порой приносит больше проблем, чем пользы. F-строки обычно надёжнее. В части 4 вы узнаете больше о полезных возможностях f-строк, когда речь идёт о форматировании вывода.

<text-box variant="hint" name="F-строки и версии Python">

Если вы используете старую версию Python, f-строки могут не работать. Они были добавлены в Python 3.6. Позже на курсе вы установите Python на свой компьютер. К сожалению, на старых операционных системах современные версии Python доступны не всегда. Если это ваш случай, то в упражнениях, где требуется f-строка, вы всегда можете попробовать решение в шаблонах упражнений прямо в браузере в этих ранних частях курса.

</text-box>

## Числа с плавающей точкой (float) {#floating-point-numbers}

Термин `floating point number` или _float_ часто встречается в программировании. Он обозначает числа с десятичной точкой. С ними можно работать почти так же, как и с целыми.

Следующая программа вычисляет среднее значение трёх чисел с плавающей точкой:

```python
number1 = 2.5
number2 = -1.25
number3 = 3.62

mean = (number1 + number2 + number3) / 3
print(f"Mean: {mean}")
```

<sample-output>

Mean: 1.6233333333333333

</sample-output>

```python
x = 27
y = 15
```

Допишите программу так, чтобы она также печатала следующее:

27 + 15 = 42

Программа должна работать корректно, даже если значения переменных изменятся. То есть, если первые две строки заменить на

```python
x = 4
```

программа должна напечатать:

<sample-output>

<in-browser-programming-exercise name="Исправьте код: Печать в одну строку" anchor="Fix the code: Print a single line" tmcname="part01-12_print_a_single_line">

Обычно каждая команда `print` печатает свою строку (включая перевод строки в конце). Но если `print` передать дополнительный аргумент `end = ""`, то перевод строки напечатан не будет.

Например:

```python
print("Hi ", end="")
print("there!")
```

<sample-output>

Hi there!

</sample-output>

Исправьте программу так, чтобы всё вычисление вместе с результатом было напечатано в одной строке. Не меняйте количество используемых команд `print`.

```python

print(5)
print(" + ")
print(8)
print(" - ")
print(4)
print(" = ")
print(5 + 8 - 4)
```

</in-browser-programming-exercise>
