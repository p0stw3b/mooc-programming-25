## Инструкция (statement) {#statement}

_Инструкция_ (statement) — это фрагмент программы, который что-то выполняет. Часто (но не всегда) это одна команда.

Например, `print("Hi!")` — инструкция, которая печатает строку текста. Аналогично, `number = 2` — инструкция, присваивающая значение переменной.

Инструкция может быть и более сложной. Она может, например, содержать другие инструкции. Следующая инструкция занимает три строки:

```python
if name == "Anna":
    print("Hi!")
    number = 2
```

В этом примере внутри условной конструкции находятся две инструкции (печать и присваивание).

## Блок {#block}

_Блок_ — это группа последовательных инструкций, находящихся на одном уровне в структуре программы. Например, блок условной конструкции содержит инструкции, которые выполняются только если условие истинно.

```python
if age > 17:
    # начало блока условной конструкции
    print("You are of age!")
    age = age + 1
    print("You are now one year older...")
    # конец блока условной конструкции

print("This here belongs to another block")
```

В Python блоки обозначаются отступом: весь код внутри блока сдвинут вправо на одинаковое количество пробелов.

Примечание: основной блок программы на Python всегда должен начинаться с самого левого края файла, без отступов:

```python
# эта программа не будет работать, потому что она не начинается с самого левого края файла
  print("hello world")
  print("this program is not very good...")
```

## Выражение {#expression}

_Выражение_ — это фрагмент кода, который при вычислении даёт значение некоторого типа данных. Во время выполнения программы выражение вычисляется, и его результат затем можно использовать в программе.

Несколько примеров выражений:

| Выражение | Значение | Тип | Тип данных в Python |
|-----------|----------|-----|--------------------|
|`2 + 4 + 3` | `9` | целое число | `int` |
|`"abc" + "de"` | `"abcde"` | строка | `str` |
|`11 / 2` | `5.5` | число с плавающей точкой | `float` |
|`2 * 5 > 9` | `True` | логическое значение | `bool` |

Поскольку у каждого выражения есть тип, результат выражения можно присваивать переменным:

```python
# переменной x присваивается значение выражения 1 + 2
x = 1 + 2
```

Простые выражения можно объединять в более сложные, например с помощью арифметических операций:

```python
# переменной y присваивается значение выражения «3 раза x плюс x в квадрате»
y = 3 * x + x**2
```

## Функция {#function}

_Функция_ выполняет некоторое действие. Функции могут принимать один или несколько _аргументов_ — данные, которые передаются функции и обрабатываются ею. Аргументы иногда также называют _параметрами_. Технически между «аргументом» и «параметром» есть разница, но на практике эти слова часто используют как взаимозаменяемые. Пока достаточно помнить, что оба термина связаны с идеей передачи данных в функцию.

Функция выполняется, когда её _вызывают_. То есть когда имя функции (и её аргументы, если они есть) встречаются в коде. Следующая инструкция вызывает функцию `print` с аргументом `"this is an argument"`:

```python
print("this is an argument")
```

Другая функция, которую вы уже часто использовали, — `input`, она запрашивает ввод у пользователя. Аргумент этой функции — сообщение, которое показывается пользователю:

```python
name = input("Please type in your name: ")
```

В этом случае функция также _возвращает_ значение. После выполнения функции место в коде, где она была вызвана, можно мысленно заменить на возвращаемое значение — это ещё одно выражение, которое было вычислено. Функция `input` возвращает строку с тем, что пользователь ввёл. Возвращаемое значение часто сохраняют в переменной, чтобы использовать его позже в программе.

## Тип данных {#data-type}

_Тип данных_ описывает свойства значения в программе. В следующем примере тип данных переменной `name` — строка (`str`), а тип данных переменной `result` — целое число (`int`):

```python
name = "Anna"
result = 100
```

Чтобы узнать тип данных любого выражения, можно использовать функцию `type`. Пример:

```python
print(type("Anna"))
print(type(100))
```

<sample-output>

<class 'str'>
<class 'int'>

</sample-output>

## Синтаксис {#syntax}

Как и в естественных языках, _синтаксис_ языка программирования определяет, как должен быть написан код. У каждого языка программирования — свой синтаксис.

Синтаксис Python, среди прочего, требует, чтобы первая строка инструкции `if` заканчивалась двоеточием, а блок внутри имел отступ:

```python
if name == "Anna":
    print("Hi!")
```

Если не соблюдать синтаксические правила, возникнет ошибка:

```python
if name == "Anna"
    print("Hi!")
```

<sample-output>

<pre>
  File "test.py", line 1
    if name == "Anna"
                    ^
SyntaxError: invalid syntax
</pre>


</sample-output>

## Отладка {#debugging}

Если синтаксис программы корректен, но программа всё равно работает не так, как задумано, значит в программе есть _ошибка_ (bug).

Ошибки проявляются по-разному. Некоторые приводят к ошибке во время выполнения. Например, программа

```python
x = 10
y = 0
result = x / y

print(f"{x} divided by {y} is {result}")
```

вызывает такую ошибку:

<sample-output>

<pre>
ZeroDivisionError: integer division or modulo by zero on line 3
</pre>

</sample-output>

Проблема здесь математическая: деление на ноль запрещено, поэтому выполнение программы прерывается.

Ошибки выполнения обычно сравнительно легко исправлять, потому что сообщение об ошибке указывает строку кода, вызвавшую сбой. Конечно, настоящая причина может находиться и в другом месте, а не именно в указанной строке.

Иногда ошибка проявляется иначе: программа выполняется, но выдаёт неверный результат. Найти и локализовать такую ошибку бывает сложно. В упражнениях этого курса тесты чаще всего как раз и предназначены для выявления подобных ошибок. Прежде чем исправлять ошибку, нужно выяснить её причину.

Процесс поиска причин ошибок называют _отладкой_ (debugging). Это крайне важный навык. Профессиональные разработчики нередко тратят на отладку больше времени, чем на написание нового кода.

Простой и эффективный способ отладки — добавлять в код диагностические выводы с помощью `print`. Проверка промежуточных результатов даёт быстрый ответ, делает ли код то, что вы ожидаете.

Ниже — попытка решить одно из упражнений из [предыдущего раздела](/ru/part-1/5-conditional-statements):

```python
hourly_wage = float(input("Hourly wage: "))
hours = int(input("Hours worked: "))
day = input("Day of the week: ")

daily_wages = hourly_wage * hours
if day == "sunday":
    daily_wages * 2

print(f"Daily wages: {daily_wages} euros")
```

Программа работает неправильно. При запуске тестов появляется следующее сообщение:

<sample-output>

<pre>
FAIL: PythonEditorTest: test_sunday_1

With input 20.0,6,Sunday correct wage 240.0 is not found in output Daily wages: 120.0 euros
</pre>

</sample-output>

При отладке упражнений на этом курсе первым шагом часто бывает проверка поведения программы на входных данных из упавшего теста. Действительно, результат не совпадает с ожидаемым:

<sample-output>

Daily wages: 120.0 euros

</sample-output>

Отладка обычно означает, что программу придётся запускать много раз. Иногда удобно временно подставить проблемный ввод прямо в код (жёстко задать значения), чтобы не вводить его вручную при каждом запуске. В нашем случае это может выглядеть так:

```python
# hourly_wage = float(input("Hourly wage: "))
# hours = int(input("Hours worked: "))
# day = input("Day of the week: ")
hourly_wage = 20.0
hours = 6
day = "Sunday"

daily_wages = hourly_wage * hours
if day == "sunday":
    daily_wages * 2

print(f"Daily wages: {daily_wages} euros")
```

Следующий шаг — добавить _отладочные сообщения_. Проблемная часть находится в обработке воскресенья, поэтому добавим `print` перед и после строки, которая должна удваивать зарплату по воскресеньям:

```python
# ...

daily_wages = hourly_wage * hours
if day == "sunday":
    print("wages before:", daily_wages)
    daily_wages * 2
    print("wages after doubling:", daily_wages)

print(f"Daily wages: {daily_wages} euros")
```

Запуск программы ничего не показывает — отладочные строки вообще не выводятся. Значит, код внутри блока `if` не выполняется, и проблема, вероятно, в условии. Давайте выведем значение булевого выражения:

```python
# ...

daily_wages = hourly_wage * hours
print("condition:", day == "sunday")
if day == "sunday":
    print("wages before:", daily_wages)
    daily_wages * 2
    print("wages after doubling:", daily_wages)

print(f"Daily wages: {daily_wages} euros")
```

Действительно, значение — `False`, поэтому содержимое блока `if` никогда не выполняется:

<sample-output>

condition:  False
Daily wages: 120.0 euros

</sample-output>

Значит, проблема в самом условии. Как часто бывает в программировании, при сравнении важен регистр букв. Обратите внимание: в выражении используется `"sunday"` без заглавной буквы, а во входных данных было `"Sunday"`. Исправим это (и в `print`, и в `if`):

```python
# ...

daily_wages = hourly_wage * hours
print("condition:", day == "Sunday")
if day == "Sunday":
    print("wages before:", daily_wages)
    daily_wages * 2
    print("wages after doubling:", daily_wages)

print(f"Daily wages: {daily_wages} euros")
```

Теперь программа печатает:

<sample-output>

condition: True
wages before: 120
wages after doubling: 120
Daily wages: 120.0 euros

</sample-output>

В начале значение `daily_wages` корректно: `hourly_wage = 20.0`, `hours = 6`, и 20.0 * 6 = 120.0. Но команда, которая должна удвоить сумму, не делает этого — значит, проблема в ней. И действительно, команда

```python
daily_wages * 2
```

вычисляет удвоенное значение, но никуда его не сохраняет. Исправим так, чтобы новое значение сохранялось:

```python
daily_wages *= 2
```

Запустим ещё раз — теперь итоговый вывод тоже верный:

<sample-output>

condition: True
wages before: 120
wages after doubling: 240
Daily wages: 240.0 euros

</sample-output>

Когда программа исправлена, не забудьте удалить все отладочные сообщения `print` и другой временный код, добавленный для отладки.

Этот пример довольно простой: в такой короткой программе ошибки, вероятно, можно найти, просто внимательно прочитав код. Однако отладочные сообщения `print` часто помогают быстро понять, где примерно находится проблема. С их помощью можно убедиться, какие части программы работают правильно, и сосредоточить поиск ошибки на наиболее подозрительных местах.

Отладочные сообщения `print` — лишь один из инструментов. Мы ещё вернёмся к теме отладки позже в курсе. Но уже сейчас полезно выработать привычку использовать `print` для поиска ошибок. Профессионалы без этого не обходятся, поэтому новичкам такой приём тоже очень помогает.

Примеры:

```python
word = "abcd"
print(len(word))

print(len("hi there"))

word2 = "howdydoody"

empty_string = ""
length = len(empty_string)
print(length)
```

<in-browser-programming-exercise name="Приведение типов" anchor="Typecasting" tmcname="part02-03_typecasting">

При программировании на Python часто нужно менять тип данных значения. Например, число с плавающей точкой можно преобразовать в целое с помощью функции `int`:

```python

temperature = float(input("Please type in a temperature: "))

print("The temperature is", temperature)

print("...and rounded down it is", int(temperature))

```

<sample-output>

Please type in a temperature: **5.15**
The temperature is 5.15
...and rounded down it is 5

</sample-output>

Обратите внимание: функция всегда округляет вниз, а не по обычным математическим правилам округления. Это пример функции округления вниз (floor).

<sample-output>

Please type in a temperature: **8.99**
The temperature is 8.99
...and rounded down it is 8

</sample-output>

Напишите программу, которая просит пользователя ввести число с плавающей точкой, а затем выводит отдельно целую часть и дробную часть. Используйте функцию `int`.

Можно считать, что введённое число всегда больше нуля.

Пример ожидаемого поведения:

<sample-output>

Please type in a number: **1.34**
Integer part: 1
Decimal part: 0.34

</sample-output>
