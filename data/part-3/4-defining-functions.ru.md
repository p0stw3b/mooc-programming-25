## Определение функции {#the-function-definition}

Прежде чем функцию можно использовать, её нужно определить. Любое определение функции начинается с ключевого слова `def` (сокращение от _define_). Затем идёт _имя_ функции, далее круглые скобки и двоеточие. Это называется _заголовком_ функции. После него, с отступом, как у блоков `while` и `if`, следует _тело_ функции.

Например, следующий код определяет функцию `message`:

```python
def message():
    print("This is my very own function!")
```

Если выполнить программу выше, кажется, что ничего не происходит. Это потому, что код в теле функции выполняется только тогда, когда функция _вызывается_.

Вызвать функцию просто: достаточно написать её имя в коде. Если добавить вызов функции в конец программы, вот так:

```python
def message():
    print("This is my very own function!")

message()
```
то будет напечатано:

<sample-output>

This is my very own function!

</sample-output>

После определения функцию можно вызывать многократно:

```python
def message():
    print("This is my very own function!")

message()
message()
message()
```

<sample-output>

This is my very own function!
This is my very own function!
This is my very own function!

</sample-output>

<text-box variant='hint' name='Тестирование собственных функций'>

Примечание: начиная с этого момента большинство упражнений на курсе будут просить вас написать собственную функцию (или функции).

Когда программа состоит только из функций, при запуске может показаться, что ничего не происходит. Следующий код ничего не выводит, хотя в нём есть `print`:

```python
def greet():
    print("Hi!")
```

Ничего не печатается, потому что код внутри тела функции `greet` выполняется только при её вызове.

«Основная» часть программы (main), которая находится ниже определений функций, должна содержать нужные вызовы функций, чтобы программу можно было протестировать. Вообще Python считает весь код вне определений функций частью _главной функции_ (main), которая выполняется при запуске файла. Поэтому добавим вызов:

```python
def greet():
    print("Hi!")

# Весь код вне определений функций — это часть
# «главной» части программы (main)
# Вызовем нашу функцию:

greet()
```

**Важно:** на этом курсе автоматические тесты требуют, чтобы «main» был пустым. В основной части решения не должно оставаться никаких команд. То есть весь код, который вы используете для тестирования, должен находиться в специальном блоке `if`:

```python
def greet():
    print("Hi!")

# Пишите код для тестирования внутри блока, как здесь:
if __name__ == "__main__":
    greet()
```

Любой код, оставленный вне блока выше, вызовет ошибку:

<img src="3_4_1.png">

Цель этого правила — чтобы решение тестировалось «на чистом листе», так как тесты часто проверяют, что печатают ваши функции. Важно помнить: тесты не выполняют код внутри блока `if __name__ == "__main__"`, поэтому код, необходимый для выполнения требований упражнения, нельзя помещать в этот блок.

</text-box>

## Аргументы функций {#function-arguments}

Функции часто принимают один или несколько _аргументов_, которые могут влиять на то, что делает функция. Например, встроенные функции Python `print` и `input` принимают в качестве аргументов текст, который нужно показать:

```python
print("Hi!")                           # аргумент — строка "Hi!"
name = input("What is your name? ")    # аргумент — строка "What is your name? "
print(name)                            # аргумент — значение переменной name
```

Мы уже упоминали, что термины _аргумент_ и _параметр_ часто используют как синонимы. Различие такое: _аргумент_ — это данные, которые передаются функции при вызове; _внутри_ функции эти данные попадают в переменные, которые называются _параметрами_. То есть, грубо говоря, при вызове мы передаём аргументы, а при определении функции мы описываем параметры.

Это может показаться чисто терминологической придиркой, и, чтобы было ещё сложнее, не все источники придерживаются такого определения. Но на этом курсе мы стараемся сохранять различие, потому что правильная терминология поможет вам понимать и другие материалы.

Определим несколько функций, которые принимают аргументы. В определении функции параметры записываются в круглых скобках после имени функции:

```python
def hello(target):
    print("Hello", target)
```

Вызовем эту функцию дважды:

```python
hello("Emily")
hello("world!")
```

и получим два разных приветствия:

<sample-output>

Hello Emily
Hello world!

</sample-output>

Посмотрим внимательнее на определение:

```python
def hello(target):
    print("Hello", target)
```

В первой строке, в заголовке функции, мы указали, что функция принимает один аргумент и записывает его в параметр с именем `target`. В теле функции команда `print` использует значение, хранящееся в `target`.

При вызове функции параметр `target` получает значение, переданное аргументом. Например, следующий вызов

```python
name = "Alan"
hello(name)
```

приводит к тому, что параметр `target` получает значение `"Alan"`.

Имена функций и их параметров подчиняются тем же принципам, что и имена переменных: они должны быть понятными и в основном состоять из строчных букв и символов подчёркивания. Конечно, есть исключения, но пока мы их игнорируем.

<text-box variant='hint' name='Тестирование функций с аргументами'>

Если ваша функция принимает один или несколько аргументов, рекомендуется тестировать её на разных значениях.

Особое внимание уделяйте «краевым случаям» для типа аргумента, с которым вы работаете. Что будет, если аргумент — 0 или отрицательное число? А если вместо целого числа придёт число с плавающей точкой? Что произойдёт, если аргумент — пустая строка?

## Ещё примеры {#more-examples}

Рассмотрим ещё несколько примеров функций с аргументами. В следующем определении параметр — это число:

```python
def squared(x):
    print(f"The square of the number {x} is {x * x}")

squared(2)
squared(5)
```

<sample-output>

The square of the number 2 is 4
The square of the number 5 is 25

</sample-output>

А в этом определении внутри тела функции есть условная конструкция `if`:

```python
def hello(name):
    if name == "Emily":
        print("Hello", name)
    else:
        print("Hi", name)

hello("Emily")
hello("Mark")
```

<sample-output>

Hello Emily
Hi Mark

</sample-output>

Эта функция принимает два аргумента:

```python
def sum(x, y):
    result = x + y
    print(f"The sum of the arguments {x} and {y} is {result}")

sum(1, 2)
sum(5, 24)
```

<sample-output>

The sum of the arguments 1 and 2 is 3
The sum of the arguments 5 and 24 is 29

</sample-output>

В функции есть вспомогательная переменная `result`, в которую сохраняется сумма аргументов.

Обратите внимание: имена параметров внутри определения функции не связаны с какими-либо переменными вне функции. Мы могли бы вызвать эту функцию и так:

```python
x = 100
y = 30
sum(1, 2)
sum(x + y, 10)
```

Это должно вывести:

<sample-output>

The sum of the arguments 1 and 2 is 3
The sum of the arguments 130 and 10 is 140

</sample-output>

В первом вызове параметры получают значения `x = 1` и `y = 2`. Во втором вызове параметры получают значения `x = 130` и `y = 10`, независимо от того, что во внешнем коде есть переменные с похожими именами.

## Внимание: использование глобальных переменных внутри функций {#warning-using-global-variables-within-functions}

В примерах выше мы увидели, что внутри функций можно создавать новые переменные. Функция также «видит» переменные, определённые вне неё — в основной части программы. Такие переменные называют _глобальными_.

Использовать глобальные переменные внутри функций обычно плохая идея. Среди прочего это может приводить к ошибкам, которые трудно отследить.

Ниже пример функции, которая «по ошибке» использует глобальную переменную:

```python
# это глобальная переменная
name = "Betty"

def hello(given_name):
    # по ошибке используем глобальную переменную вместо параметра
    print("Hello", name)

hello("Steve")
hello("Betty")
```

<sample-output>

Hello Betty
Hello Betty

</sample-output>

Как бы мы ни вызывали функцию с разными аргументами, она всегда будет печатать значение `"Betty"`, хранящееся в глобальной переменной.
