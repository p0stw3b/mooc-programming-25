До этого момента все упражнения на курсе выполнялись прямо на страницах курса — во встроенных окнах редактора. Программирование в браузере отлично подходит для самых первых шагов, но теперь пришло время перейти на отдельный редактор, предназначенный для программирования.

Существует множество редакторов, подходящих для программирования. На этом курсе мы будем использовать [Visual Studio Code](https://code.visualstudio.com/) — редактор, который стал очень популярным в последние годы.

Пожалуйста, установите Visual Studio Code на свой компьютер. Возможно, также потребуется установить Python и расширение Visual Studio Code для Python. Вам также понадобится плагин TMC — он отвечает за запуск тестов для упражнений. В плагине TMC выберите организацию **MOOC** и курс **Python Programming 2025**.

[Здесь находится инструкция](https://www.mooc.fi/en/installation/vscode) по установке и запуску всего необходимого. Прочитайте, как работать с упражнениями и как отправлять решения, а затем выполните задание ниже:

<programming-exercise name='Привет, Visual Studio Code' anchor="Hello Visual Studio Code" tmcname='part04-01_hello_visual_studio_code'>

Напишите программу, которая спрашивает у пользователя, каким редактором он пользуется. Программа должна продолжать спрашивать до тех пор, пока пользователь не введёт _Visual Studio Code_.

Посмотрите пример ожидаемого поведения:

<sample-output>

Editor: **Emacs**
not good
Editor: **Vim**
not good
Editor: **Word**
awful
Editor: **Atom**
not good
Editor: **Visual Studio Code**
an excellent choice!

</sample-output>

Если пользователь вводит Word или Notepad, программа должна отвечать _awful_. Для всех остальных «плохих» редакторов ответ должен быть _not good_.

Реакция программы должна быть нечувствительной к регистру. То есть один и тот же ввод в нижнем, верхнем или смешанном регистре должен приводить к одинаковому результату:

<sample-output>

Editor: **NOTEPAD**
awful
Editor: **viSUal STudiO cODe**
an excellent choice!

</sample-output>

Подсказка: проще всего привести строку к одному регистру. Строковый метод Python `lower` переводит строку полностью в нижний регистр. Пример:

```python
mystring = "Visual Studio CODE"
if "visual studio code" == mystring.lower():
    print("this was the string I was looking for!")
```

**Важно:** в этом упражнении не просят писать функции, поэтому вы **не должны** помещать код внутрь блока `if __name__ == "__main__"`. То же относится ко всем последующим упражнениям, где явно не сказано писать функции.

</programming-exercise>

## Запуск кода {#executing-code}

В Visual Studio Code проще всего запускать код, нажимая на треугольник в правом верхнем углу. Иногда программа может остаться выполняться в фоне — например, ждать ввода пользователя или застрять в бесконечном цикле — и вы можете этого не заметить. Тогда следующая программа не запускается, потому что предыдущий процесс всё ещё занимает ресурсы. Быстрый способ остановить выполнение — нажать _Control_+_C_. После этого следующий запуск должен пройти нормально.

## Интерактивный интерпретатор Python {#the-interactive-python-interpreter}

Один из важнейших инструментов для любого Python-разработчика — интерактивный интерпретатор Python.

Запуск интерпретатора зависит от платформы. В Linux или macOS обычно достаточно набрать `python3` в терминале. В Windows команда может быть `python`. Запуск интерпретатора на Mac выглядит так:

<img src="4_1_1.png">

Интерпретатор можно запустить и внутри Visual Studio Code. Сначала запустите программу, нажав на треугольник — это откроет вкладку _Terminal_, где можно набрать `python3` (или `python`):

<img src="4_1_2.png">

Можно также попробовать интерпретатор Python прямо в браузере, например <https://www.python.org/shell/>.

Интерпретатор позволяет выполнять код построчно — сразу после того, как вы его вводите. Когда вы пишете строку кода и нажимаете _Enter_, интерпретатор тут же выполняет эту строку и показывает результат:

<img src="4_1_3.png">

Любой код Python, который можно написать в файле, можно писать и в интерпретаторе. Можно присваивать значения переменным и даже определять функции:

```python
>>> t = [1,2,3,4,5]
>>> for number in t:
...   print(number)
...
1
2
3
4
5
>>> def absolute_value(number):
...   if number<0:
...      number = -number
...   return number
...
>>> x = 10
>>> y = -7
>>> absolute_value(number)
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
NameError: name 'number' is not defined
>>> absolute_value(x)
10
>>> absolute_value(y)
7
>>>
```

Интерпретатор лучше всего подходит для небольших проверок. Например, в нём можно попробовать функции и методы или убедиться, что они вообще существуют:

```python
>>> "TextYteXt".toupper()
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
AttributeError: 'str' object has no attribute 'toupper'
>>> "TextYteXt".upper()
'TEXTYTEXT'
>>>
```

Если вам нужен метод, и вы _почти_ помните его название, иногда быстрее не идти в Google, а использовать в интерпретаторе функцию `dir`. Она показывает, какие методы доступны у данного объекта:

У строк в Python очень много методов, как видно выше. На этом этапе можно пока игнорировать методы с подчёркиваниями в имени, а остальные вполне могут пригодиться. Часть из них можно понять, просто попробовав, а остальное легко найти в интернете.

Давайте попробуем несколько — `reverse` и `clear` выглядят полезно:

```python
>>> numbers = [1,2,3,4,5]
>>> numbers.reverse()
>>> numbers
[5, 4, 3, 2, 1]
>>> numbers.clear()
>>> numbers
[]
```

Как видите, эти методы делают примерно то, что можно предположить по их названиям.

Обратите внимание: интерпретатор ничего не печатает, когда вы выполняете `numbers.reverse()`. Это потому, что интерпретатор печатает результат только если строка кода имеет значение. Метод `reverse()` здесь ничего не возвращает.

В примере выше мы выводили значение списка `numbers`, просто набирая имя переменной. На самом деле в интерпретаторе редко нужно явно писать `print` — но, если хотите, можно.

Не забывайте закрывать интерпретатор, когда закончили. Команды `quit()` или `exit()` закроют его, как и сочетание клавиш _Control_+_D_ (Linux/Mac) или _Control_+_Z_ (Windows). В Visual Studio Code это особенно важно: если попытаться запустить другую Python-программу, пока интерпретатор ещё работает, может появиться довольно непонятное сообщение об ошибке:

## Встроенный отладчик {#the-built-in-debugger}

Мы уже потратили немало времени на отладку, в основном используя диагностические `print`. В Visual Studio Code есть ещё один инструмент: встроенный _визуальный отладчик_.

Чтобы начать отладку, сначала нужно поставить _точку останова_ (breakpoint) в коде. Это место, где отладчик остановит выполнение программы. Точку останова можно поставить, щёлкнув по левому полю возле нужной строки.

Следующий пример — немного ошибочная попытка решить упражнение _The sum of consecutive numbers_ из [предыдущей части](/ru/part-3/1-loops-with-conditions). Точка останова стоит на строке 5:

<img src="4_1_5.png">

После установки точки останова выберите _Start debugging_ в меню _Run_. Откроется список вариантов, где нужно выбрать _Python File_:

<img src="4_1_6.png">

Так запускается отладчик: он выполняет код как обычно, пока не дойдёт до точки останова и не остановится. Если программа запрашивает ввод, не забудьте вводить его в терминале:

<img src="4_1_7.png">

Слева появляется панель _Variables_ с текущими значениями всех переменных, активных в данной точке. Можно продолжать выполнение построчно, нажимая на стрелку вниз с подписью _Step into_.

На изображении ниже цикл в коде уже повторился несколько раз:

<img src="4_1_8.png">

У отладчика есть вкладка _Debug console_, которая позволяет вычислять выражения, используя текущие значения переменных. Например, можно проверить значение булевого выражения в условии цикла:

<img src="4_1_9.png">

В программе можно поставить несколько точек останова. Когда выполнение остановлено, его можно продолжить, нажав на синий треугольник. Выполнение будет идти до следующей точки останова.

Встроенный визуальный отладчик — хорошая альтернатива отладочным `print`. Вам решать, чем чаще пользоваться дальше. У каждого программиста свои предпочтения, но всегда полезно попробовать разные подходы, прежде чем «привязываться» к одному.
